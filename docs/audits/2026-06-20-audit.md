# Repository audit — 2026-06-20

Self-directed ~20-pass audit per `AUDIT-BRIEF.md`. Every finding is built from
primary sources (code / command output / GitHub API), not from prior
`progress.md` summaries. Evidence is cited as `file:line` or command output.
**Audit only — nothing was refactored.** Branch: `claude/audit-brief-execution-xur28z`
(== `origin/main`, 0 ahead / 0 behind at audit time).

## How this was done

Phase 0 mapped all 265 tracked files and read every load-bearing file directly
(CLAUDE.md + @imports, all `.claude/rules/*`, all 10 hooks, `settings.json`, the
full `src/` app, the GL/audio/shader code, the build/verify tooling) plus four
parallel `Explore` sweeps over `tools/`, `test/`, `.claude/skills/`, and
`docs/`+`research/`. Phase 1 chose tools from what surfaced: `git`, the local
gates (`npm run health`, smoke, gen-docs `--check`, rag `--check`), and the
GitHub MCP (CI run conclusions, repo visibility, PR/branch state). Phase 2 ran
the lenses below.

---

## Verdict (one line)

The **app code is in good shape and largely rule-compliant**, but the **repo
plumbing is broken in three load-bearing ways**: CI is red on `main`, the repo
is public while every doc assumes it is private, and the auto-deploy trigger
points at a deleted branch. None of these is visible from the narrative docs —
they only show up against primary sources.

# CRITICAL

## C1 — CI `verify` is RED on `main` and cannot self-heal in-container
**Evidence:**
- GitHub Actions: the three most recent `verify` runs on `main` are all
  `conclusion: failure` — sha `05d07af`, `622813c`, `10f2462` (run list,
  2026-06-20). First failure = `10f2462` "regen ENCYCLOPEDIA for progress.md
  milestone".
- Local: `npm run health` → exit 1, sole failing gate `RAG index drift gate:
  index.json stale — run: npm run rag:index` (`tools/health.mjs:21`).
- Root cause: `tools/rag/chunk.mjs:58-63` builds the corpus from **every
  git-tracked `.md`** except `research/corpus/` + `research/fmhy-dev-tools/`
  (`tools/mcp/lib/docs.mjs:18-23`). Commits `aa33d8d` and `10f2462` changed
  `progress.md` / `ENCYCLOPEDIA.md` (both in the corpus) but **did not rebuild**
  `tools/rag/index.json` (last rebuilt at `aa78044`/`04f8ef5`). The committed
  `builtFromHash` (`tools/rag/build-index.mjs:50-56`) no longer matches.
- It cannot be fixed from this container: the embedder `@huggingface/transformers`
  is absent from the stale snapshot (orient hook env-check; `.claude/hooks/orient.sh:84-88`),
  so `npm run rag:index` can't run here.

**Impact:** `verify.yml` runs the RAG `--check` step *before* `npm ci` /
Playwright / render-check / shader-validate / MCP selftest / rag-tests
(`.github/workflows/verify.yml:41-59`). A failed step fails the job and **skips
all later steps** — so while the index is stale, **none of the real verification
runs on any push.** This is the single most important finding: the project's
safety net is currently off.

**Fix:** rebuild + commit `index.json` on a machine that has the embedder
(`npm ci && npm run rag:index`), or merge PR #5's `session-start.sh` which
auto-installs the toolchain (see I5). Consider making the RAG `--check` a
non-blocking/`continue-on-error` step, or moving it after render-check, so a
stale index never masks the rest of CI. Longer-term, the recurring
"edit a doc → forget to rebuild the index" footgun (already logged in
`.claude/rules/gotchas.md`) argues for a `gen-docs`-style PostToolUse rebuild or
a much cheaper structural index.

## C2 — The repository is PUBLIC, but every doc is written assuming it is PRIVATE
**Evidence:**
- GitHub API: `aetherrigstudio-art/Primordial-viz` → `"private": false`,
  `"visibility": "public"` (repo search, 2026-06-20).
- Contradicting assumptions, stated as the safety net for the privacy posture:
  - `.claude/rules/deploy.md` (Client-side privacy): *"The repo is **private**;
    commits carry `Co-Authored-By: Claude` trailers, so history would reveal AI
    authorship — just don't make it public without a scrub."*
  - `progress.md` (privacy session): *"**Note:** repo is private … don't make it
    public without scrubbing history."*

**What is now publicly exposed:** the full 72 KB `progress.md` internal session
log; `docs/prompts/claude-opus-4-8-system-prompt.md` +
`docs/prompts/system-prompt-ingest.md` (a complete assistant **system prompt**
and its integration map); the entire `.claude/` agent tooling and rules; and the
whole git history with `Co-Authored-By: Claude …` trailers on most commits.

**Impact:** the deployed-site privacy rule explicitly *relaxes* everything
non-served on the grounds that "the repo is private." That premise is false, so
the relaxation is unjustified — anyone can read the AI-authorship trail, the
internal narrative, and the ingested system prompt. This is a
decision-for-the-operator, not a code bug: either (a) flip the repo back to
private, or (b) consciously accept public exposure and scrub
history/`docs/prompts/` accordingly. It should not stay in the current
silent-mismatch state.

## C3 — Auto-deploy is broken: `deploy.yml` triggers on a branch that no longer exists
**Evidence:**
- `.github/workflows/deploy.yml:13-15` → `on: push: branches:
  [claude/review-claude-md-di5jvm]`.
- That branch is gone: `git branch -a | grep review-claude-md-di5jvm` → no match
  (it was PR #2's head, merged + deleted). Current branches: `main`, `audit-…`,
  `hooks-*`, `check-main-*`, `orient-*`, `design/*`.
- Contradicting claims of working auto-deploy:
  - `CLAUDE.md`: *"Web deploy is **automated**: push → GitHub Actions FTPS →
    `https://primordial.video/Test/`."*
  - `.claude/hooks/orient.sh:74`: *"AUTO-deploys on push (GitHub Actions FTPS …)."*

**Impact:** pushing to `main` (or any active branch) does **not** redeploy the
live `/Test/` preview. Only the manual `workflow_dispatch` path works. The live
site silently drifts from `main`. Fix: point the push trigger at `main` (and/or
the working branch), or delete the dead push trigger and document that deploy is
manual-dispatch only.

# IMPORTANT

## I1 — Licensing contradiction: MIT + public vs "proprietary commercial" shaders
**Evidence:** `LICENSE:1-3` = MIT, `© 2026 Primordial Studio LLC`; `README.md:88`
"MIT"; GitHub `license.key = "mit"`. But `.claude/rules/shaders.md:10-14` frames
the whole write-our-own rule around protecting *"a proprietary paid tool,"* and
`CLAUDE.md` calls the shaders "commercial work."

**Impact:** an MIT license grants everyone the right to "use, copy, modify,
merge, publish, distribute, sublicense, and/or **sell**" — including the shaders
the rules work hardest to keep proprietary. With the repo also public (C2), the
shaders are effectively open source. The "don't copy NC/SA Shadertoy code"
hygiene is still correct (avoids *inbound* copyleft), but the *outbound*
proprietary stance is contradicted by the chosen license. Decide intentionally:
keep MIT (then the "proprietary" framing in the rules is wrong and should be
softened) or relicense to proprietary/all-rights-reserved (then MIT must go).
