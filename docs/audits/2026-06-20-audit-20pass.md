## Pass 10 — Security
- No secrets committed (`.env.example` placeholders only; `npm run audit` clean).
- `guard.mjs` destructive-command coverage is solid and unit-tested.
- **[NEW] `tools/mcp/lib/looks.mjs:83` + `tools/gen-docs.mjs:377` — `$`-in-
  replacement corruption.** Both do `replace(re, \`$1${body}\n$2\`)` where `body`
  is interpolated text. A `$` in look name/description (user-supplied via the MCP
  `create_look`/`update_look` tools) is reinterpreted as a `replace` pattern and
  **corrupts the committed `src/looks/registry.js`**. Real latent bug; fix is a
  function replacer `(_, a, b) => a + body + '\n' + b`.
- **[NEW] `src-tauri/tauri.conf.json:26` ships `csp: null`** — no Content-
  Security-Policy on the desktop webview (permissive default). Capabilities are
  least-privilege (`capabilities/default.json` = `core:default` only), which is good.
- **[NEW] `tools/eval-skills.mjs:230-233`** calls `messages.create({ output_config:
  { effort, format }, … })` — `output_config`/`effort`/`format` are not standard
  Messages API params; the untested tier-2/3 path would throw at runtime.
  *Verify against the pinned `@anthropic-ai/sdk` before relying on it* (I did not
  run it — no API key — so this is "shape mismatch," flagged not asserted).
- **[NEW low] `research/scripts/scrape-blog.py:76`** sends a spoofed
  `User-Agent: Mozilla/5.0`; neither scraper honors robots.txt despite the
  comment claiming to. Orphaned dev scripts; courtesy/ToS concern only.
- Low: Gemini CLI puts the key in the URL query (`sort-vision.mjs:60`); OAuth/
  Gemini error bodies are echoed in thrown errors — fine for a local dev CLI,
  not for a shared log sink. No command injection (all child-process calls use
  arg arrays, no `shell:true`); `getDoc` is allowlist-gated (`docs.mjs:58`).
